Positioning

Absolutely relative or relatively absolute?

Well, if one thing needs to be fixed it is that horrible caption. At least now I have your attention so let's get right into it!

(the) Position property

When designging a website you will be conducting a diverse list of tasks including structuring, ensuring proper semantic flow is present, scripting cool stuff and, tacked onto that, a lot of debugging (Whew!/Yay).

When you are first starting out you can be sure that a decent portion of that debugging will include trying to get the componendts of your website to get to (and stay!) in the rignt position. There are many properties you can use to position elements with CSS but one that is going to affect the way they interact with your contant is the position property.

You have 5 values for the position property: Static, Relative, Absolute, Fixed and Sticky; today we will be exploring Relative, Absolute and Fixed. They all change the way your elements interact with eachother in their own way; understanding how they work should help you get those stubborn elements to go where you want them to and in turn help you design more user friendly, responsive and better looking web pages.


Relative

Setting the position property to relative will make the element move relative to the current position of it's parent element. If you have no idea what the parent element is you can imagine it as the vessel your element is riding in.

To help visualise this relationship you can imagine that you are a little baby kangaroo (joey?) sitting in your mother's pouch (we'll name her Skippy). If Skippy starts jumping around and you stay completely still you will still move around although relative to Skippy you have remained in the same position. Likewise you can shift 5cm to the right in the pouch after Skippy stopped moving around and you will have moved 5cm to the right of your (collective) relative positioning.

In essence you can move the parent element (vessel/Skippy) around however much you want and your element will stay in the same relative position before and after the move. To illustrate this I will demonstrate with two examples using simple positional properties like left which moves an element the set value away from the left and top which does the same from the, you guessed it, top.

Parent/Skippy
Child
.parent {
    border-style: solid;
    border-width: 1px;
    height: 110px;
    width: 110px;
    margin: 20px
}
Parent/Skippy
Child
.parent {
    position: relative;
    border-style: solid;
    border-width: 1px;
    height: 110px;
    width: 110px;
    margin: 20px
}

Looking at the two examples you can see how two child elements with their position set to relative and the exact same positional properties can be in differing positions based on where their parent element is placed.

Another important moment to note is that if you do not specify position: relative; the left and top properties will not function. This is demonstrated in the styling for the Parent element with the only difference being the definition of a position property.

The relative positioning can be very useful if you for example need to move an icon or an image away from the corner of your web page or just reposition it slightly relative to its parent container.


Absolute

Marking your element with the absolute positioning will not cause an old goblin of a priestly persuasion to suggest you should look out for one anotha (/high-tier-vagueness-reference). It will however allow you to move and position your element freely without affecting any of the other elements on your page. You can also refer to this as removing the element from the normal flow of the web page.

Normally an absolutely positioned element will place itself relative to it's closest ancestor/parent and is able to overlap other elements. To visualise this you can imagine having two small boxes sitting next to eachother in a bigger box. when you set one of the boxes to absolute positioning it will immediately move to the upper left corner of the bigger box and overlap the other box. From there you can move the box around ewith the same properties as you used with the relative positioning to move the box around without affecting the other contents of the bigger box. The above example utilizes the normal document flow where a block (the child elements in this example) will be placed on the next line consecutively depending on what order they appear in the HTML file.

These following two examples utilizes absolute positioning in two slightly different ways.

.absolute-child {
    background-color: lightpink;

As you can see, the first example places the child element in front of the other child element without shifting anything else inside the parent around.

The next example places the child element in the top right corner, again without shifting or moving any other elements.

For comparison let's have a look if we replace absolute with relative for the second example:

Parent
Child
Child
.absolute-child {
position: relative;
right: 0px;
top: 0px;
}

As demonstrated the element will stick to the normal flow and order of the document and thus will not be able to move to the top right corner as it would be occupying the same line as the parent and the preceeding child element.

Absolute positioning can be quite useful for example when you want to position a "close" button in the upper right corner of an element without moving every other child of that element around. Imagine if you have a 5x5 sliding puzzle and every piece is in the right place except the bottom right and the open tile is in the upper left. With absolute positioning you can magically move your bottom right piece to the upper left and "win" the game, Yay! (Yes, yes, that analogy does not work completely in practice, but I believe it illustrates the concept well nonetheless!).


Fixed

Fixed positioning, commonly used for navigation bars works similarly to absolute positioning in the aspect that it removes your element from the normal flow of your web page. The difference lies in that an element with the fixed position will stay in the same place of the users screen regardless of where they scroll. This can be particularly useful for the aforemenetioned navigation bar or a chat functionality which can stay at the bottom of your page, both accessible and discreet at the same time. Each value for the position property has it's own uses and functinality, knowing how and when to use them is a solid fundamental skill in designing well flowing, responsive web experiences.
